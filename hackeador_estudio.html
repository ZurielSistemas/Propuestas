5 Control PD — El corazón del movimiento
▾
def PD(input1, input2):
    global lastError, PDscale
    PDscale   = (abs(power) / 100) * 0.7 + 0.3
    error     = input1 - input2
    result    = (error * kP) + ((error - lastError) * kD)
    result    = result * PDscale
    lastError = error; return result

def acceleration(movedDistance, totalDistance, targetSpeed, acc, dec):
    speedChange = abs(targetSpeed) - minSpeed
    acc_speed = speedChange * (movedDistance / accLength) + minSpeed
    dec_speed = speedChange * ((totalDistance - movedDistance) / decLength) + minSpeed
    return min(acc_speed, dec_speed, abs(targetSpeed))
P empuja hacia el objetivo. D frena para no pasarse. Juntos: llegada rápida y precisa sin oscilación.

PDscale: a velocidad 0 → 0.3 (corrección mínima). A velocidad 100 → 1.0. Evita correcciones violentas durante arranque y frenado. min() en acceleration toma siempre la curva más baja — perfil trapezoidal automático.

def move(speed, totalDistance, acc=True, dec=True):
    resetMotors(); movedDistance = 0; setPD(gyroPD, 1)
    while(movedDistance <= totalDistance):
        movedDistance = (abs(left_motor.angle()) + abs(right_motor.angle())) / 2
        power = round((abs(speed)/speed) * acceleration(...))
        getYaw(); correction = round(PD(yaw, heading))
        run(power - correction, power + correction)
    stop()

def turn(mode, change, speed=100, checking=15):
    heading = heading + change
    # mode=1: pivote izq | mode=2: pivote der | mode=3: giro en sitio
    count = 0
    while(count <= checking):
        getYaw(); correction = round(PD(yaw, heading))
        count = count+1 if abs(yaw-heading) <= 2.5 else 0
    stop()

def follow(side, speed, totalDistance, ...):
    lightFiltered = floor.reflection()
    while(movedDistance <= totalDistance):
        lightFiltered = (lightFiltered * 3 + floor.reflection()) / 4
        correction = PD(lightTarget, lightFiltered)
        if side == 2: correction *= -1
        run(power - correction, power + correction)
checking=15: el robot debe estar dentro de ±2.5° durante 15 ciclos consecutivos antes de parar. Evita que pare mientras aún tiene inercia.

def setup():

    # Sin global, solo existirían dentro de setup() y las demás funciones
    # verían: UnboundLocalError: local variable referenced before assignment
    global lightPD, junctionPD, gyroPD, minSpeed, accLength, decLength
    global heading, gyroRatio, lightTarget, lightHigh, lightLow
    global blackThreshold, whiteThreshold, correctionLimit
    global currentPosition, h, s, v, BatteryScale

    # La batería LiPo trabaja entre 7000 mV (agotada) y 8250 mV (llena).
    # (voltaje - 7000) / 1.25 / 10 mapea ese rango a 0–100%.
    # 7000mV → 0%  (BatteryScale=0.0)
    # 7625mV → 50% (BatteryScale=0.5)
    # 8250mV → 100% (BatteryScale=1.0)
    BatteryVoltage    = hub.battery.voltage()
    BatteryPercentage = round((BatteryVoltage - 7000) / 1.25) / 10
    BatteryScale      = BatteryPercentage / 100
    if BatteryScale < 0: BatteryScale = 0

    # PD constants para cada modo de movimiento
    lightPD    = [0.55, 9]   # kP, kD para seguir línea
    junctionPD = [0.45, 5]   # kP, kD para detectar cruce (más conservador)
    gyroPD     = [6, 30]     # kP, kD para movimiento recto

    # A mayor voltaje los motores giran más rápido con el mismo comando.
    # BatteryScale compensa esto ajustando velocidades y constantes automáticamente.

    # Desviado a la derecha → gira de más → bajar a ~0.98

5 Funciones utilitarias — run, stop, resetMotors
# dc() = duty cycle: potencia directa, sin PID de PyBricks.
# El programa controla todo manualmente con el PD implementado en código.
# run(100, 100)  → ambas ruedas al máximo hacia adelante
# run(-100, 100) → rueda izq atrás, rueda der adelante = giro en sitio
def run(left, right):
    left_motor.dc(left)
    right_motor.dc(right)

# brake(): freno eléctrico activo (cortocircuita las bobinas).
# Más fuerte que coast() que solo suelta el motor y lo deja parar por fricción.
# wait(duration) estabiliza el robot antes del siguiente movimiento.
def stop(duration=0):
    left_motor.brake()
    right_motor.brake()
    wait(duration)

# Resetea el encoder de cada motor a 0.
# Sin resetear al inicio de cada movimiento, los grados acumulados del movimiento
# anterior interferirían con el cálculo de distancia del movimiento actual.
def resetMotors():
    left_motor.reset_angle(0)
    right_motor.reset_angle(0)

6 El sistema PD — setPD, PD, acceleration
# kP y kD son las "ganancias" del controlador PD.
# reset=1 → reiniciar lastError para transición limpia al nuevo movimiento.
# reset=0 → mantener lastError para transición suave (ej: follow → followUntilJunction)
def setPD(PD, reset):
    global kP, kD, lastError
    kP = PD[0]; kD = PD[1]
    if reset == 1: lastError = 0

def PD(input1, input2):
    global lastError, PDscale

    # PDscale escala la corrección según velocidad actual.
    # A velocidad 0: PDscale=0.3 (corrección mínima — robot casi detenido).
    # A velocidad 100: PDscale=1.0 (corrección completa).
    # Evita que el robot "luche" con correcciones fuertes cuando va casi parado.
    PDscale = (abs(power) / 100) * 0.7 + 0.3

    error = input1 - input2   # diferencia entre objetivo y actual

    # COMPONENTE PROPORCIONAL (P): corrige proporcional al error actual.
    # Solo con P: el robot llega al objetivo pero lo sobrepasa y oscila indefinidamente.
    # COMPONENTE DERIVATIVO (D): corrige la velocidad de cambio del error.
    # Si el error está bajando rápido (robot corrigiéndose), D frena para no sobrepasar.
    # Es la amortiguación del sistema — como el amortiguador de un carro.
    result    = (error * kP) + ((error - lastError) * kD)
    result    = result * PDscale
    lastError = error          # guardar para la próxima iteración (componente D)
    return result

def acceleration(movedDistance, totalDistance, targetSpeed, acc, dec):
    speedChange = abs(targetSpeed) - minSpeed

    # acc_speed: sube de minSpeed a targetSpeed según distancia recorrida.
    if acc: acc_speed = speedChange * (movedDistance / accLength) + minSpeed
    else:   acc_speed = abs(targetSpeed)

    # dec_speed: baja de targetSpeed a minSpeed según distancia restante.
    if dec: dec_speed = speedChange * ((totalDistance - movedDistance) / decLength) + minSpeed
    else:   dec_speed = abs(targetSpeed)

    # min() siempre toma la curva más baja.
    # En distancias cortas las dos rampas se solapan — perfil triangular suave.
    # En distancias largas hace el perfil trapezoidal completo: sube, mantiene, baja.
    return min(acc_speed, dec_speed, abs(targetSpeed))
En seguimiento de línea: input1=lightTarget, input2=luz_actual. En movimiento recto: input1=yaw, input2=heading. La misma función PD sirve para ambos casos.

# move(): avanza distancia exacta corrigiendo rumbo con giroscopio.
# Promedia encoders: si el PD hace girar una rueda más, la distancia correcta
# es el promedio — no la de una sola rueda.
# 360° de rueda ≈ 19.6 cm con la rueda grande.
def move(speed, totalDistance, acc=True, dec=True):
    global power
    resetMotors(); movedDistance = 0; setPD(gyroPD, 1)
    while(movedDistance <= totalDistance):
        movedDistance = (abs(left_motor.angle()) + abs(right_motor.angle())) / 2
        power = acceleration(movedDistance, totalDistance, speed, acc, dec)
        power = round((abs(speed)/speed) * power)  # preserva signo original
        getYaw()
        correction = round(PD(yaw, heading))
        # Corrección positiva (desviado a la derecha): izq más lento, der más rápido.
        run(power - correction, power + correction)
    stop()

# turn(): tres modos de giro con PD ajustado por nivel de batería.
# mode=1: solo rueda izquierda (pivote sobre derecha, radio grande)
# mode=2: solo rueda derecha (pivote sobre izquierda, radio grande)
# mode=3: ambas en sentido contrario (giro en sitio — más rápido y preciso)
def turn(mode, change, speed=100, checking=15):
    global heading, power
    resetMotors()
    heading = heading + change  # actualizar rumbo objetivo

    # Batería llena → motores más potentes → tienden a sobrepasar el ángulo.
    # kP más bajo (menos agresivo) + kD más alto (más amortiguación).
    # Batería baja → motores débiles → tienden a quedarse cortos.
    # kP más alto + kD más bajo.
    if mode <= 2:
        setPD([15-round(3*BatteryScale), 180+round(50*BatteryScale)], 1)
    else:
        setPD([12-round(3*BatteryScale), 220+round(50*BatteryScale)], 1)

    count = 0
    while(count <= checking):
        getYaw()
        sync  = left_motor.angle() + right_motor.angle()
        sync *= 1   # ← placeholder (ver Mejora M4)
        power = 100
        correction = round(PD(yaw, heading))

        # Limitar corrección máxima para no superar la velocidad configurada.
        if abs(correction) > speed:
            correction = speed * (correction / abs(correction))

        if   mode == 1: run(-correction, 0)
        elif mode == 2: run(0, correction)
        else:           run(-correction-sync, correction-sync)

        # checking=15: robot debe estar dentro de ±2.5° durante 15 ciclos consecutivos.
        # Evita que pare mientras aún tiene inercia y está oscilando.
        count = count+1 if abs(yaw-heading) <= 2.5 else 0
    stop()

# curve(): avanza en arco — el heading objetivo cambia progresivamente.
# Usa kD a la mitad porque el heading cambia continuamente y un kD alto
# resistiría los cambios planificados del objetivo.
def curve(speed, totalDistance, change, acc=True, dec=True):
    global power, heading
    startHeading = heading; resetMotors(); movedDistance = 0
    setPD([gyroPD[0], gyroPD[1]/2], 1)
    while(movedDistance <= totalDistance):
        movedDistance = (abs(left_motor.angle()) + abs(right_motor.angle())) / 2
        power = round((abs(speed)/speed) * acceleration(...))
        heading = (movedDistance / totalDistance) * change + startHeading
        getYaw(); correction = round(PD(yaw, heading))
        run(power - correction, power + correction)
    heading = startHeading + change; stop()

# follow(): seguir el BORDE de la línea (no el centro).
# side=1: borde izquierdo (sensor a la izquierda de la línea)
# side=2: borde derecho → corrección se invierte (* -1)
# Filtro promedio móvil: cada nueva lectura tiene solo 25% de peso.
# Sin filtro, una sombra de juez o polvo causaría corrección violenta.
def follow(side, speed, totalDistance, acc=True, dec=True, limit=False, reset=False):
    global power, heading
    resetMotors(); movedDistance = 0; setPD(lightPD, 1)
    lightFiltered = floor.reflection()
    while(movedDistance <= totalDistance):
        movedDistance = (abs(left_motor.angle()) + abs(right_motor.angle())) / 2
        power = acceleration(...)
        raw = floor.reflection()
        lightFiltered = (lightFiltered * 3 + raw) / 4  # filtro promedio móvil
        correction = PD(lightTarget, lightFiltered)
        if limit and abs(correction) >= correctionLimit:
            correction = correctionLimit * (correction / abs(correction))
        if side == 2: correction *= -1
        run(power - correction, power + correction)
        wait(3)  # 3ms entre iteraciones para lectura estable
    stop()

# followUntilJunction(): sigue la línea hasta detectar un cruce (negro).
# Usa junctionPD más conservador para no pasar de largo el cruce.
# reset=0: no resetear lastError → transición suave desde follow() previo.
def followUntilJunction(side, speed):

        wait(3)

I1¿Qué hace el componente Proporcional del PD? ¿Y el Derivativo? ¿Por qué se necesitan los dos?
P: corrige el error actual (error × kP). Solo con P el robot oscila: llega al objetivo, sobrepasa, vuelve, sobrepasa otra vez.

D: corrige la velocidad de cambio del error ((error-lastError) × kD). Si el error está bajando rápido, D aplica un freno para no sobrepasar.

P lleva al objetivo. D frena para no pasarse. Juntos: llegada rápida y precisa sin oscilación.
I2¿Por qué lightTarget = promedio − 10 y no el promedio exacto?
Si el target fuera el promedio exacto, el robot seguiría el centro de la línea. Al restar 10, el punto se mueve hacia el negro — el robot "prefiere" estar sobre el lado oscuro. Más robusto ante sombras de jueces y variaciones de luz en competencia.
I3¿Qué hace GYRO_RATIO y cómo sabes si necesitas ajustarlo?
Compensa si el Hub sobreestima o subestima los ángulos reales.

Cómo verificar: haz 4 × turn(3, 90). El robot debe quedar exactamente donde empezó. Desviado a la derecha → bajar a ~0.98. A la izquierda → subir a ~1.02. Ajustar en pasos de 0.01.

junctionPD = [0.45, 5] es más conservador que lightPD = [0.55, 9] — al llegar a un cruce, una corrección agresiva podría hacer que el robot pase de largo. Además se llama con setPD(..., 0) sin resetear lastError, para transición suave desde el follow() previo.
I5¿Qué significa pause=False en backDegree() y cuándo usarías pause=True?
pause=False: el código llama run_target y continúa inmediatamente. El gripper y el robot se mueven en paralelo — ahorra tiempo.

pause=True: espera que el motor llegue a la posición antes de continuar. Usar cuando el robot necesita el gripper listo antes de avanzar al bloque.
I6¿Por qué move() promedia los dos encoders?
Si el PD está corrigiendo el rumbo, una rueda avanza más que la otra. Usar solo un encoder daría una distancia incorrecta. Al promediar (abs(left) + abs(right)) / 2, los efectos de la corrección se cancelan y el resultado refleja la distancia lineal real del centro del robot.
